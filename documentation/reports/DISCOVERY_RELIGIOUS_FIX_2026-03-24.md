# Discovery UI Refinement & Religious Category Fix Report (2026-03-24)

## Overview
This report documents the fixes and refinements made to the Discovery feature to resolve issues with the Religious category display and improve overall UI minimalism.

## Issues Identified
1. **Missing Religious Row**: The Religious category row was completely missing if the backend returned fewer than two cards due to a strict `hero && detail` conditional in the frontend.
2. **Card Loss**: If a row was hidden due to the above condition, the single card that *was* returned would also be excluded from the overflow/remaining section due to fixed slice logic (`slice(2)`).
3. **API Bottleneck**: The backend was capped at 4 total religious candidates (`ISLAMIC_API_MAX_CANDIDATES`), making it difficult to populate both "Ayet" and "Hadith" cards simultaneously.
4. **Diyanet API Failure**: The Diyanet API was returning 404 errors due to an outdated development URL.

## Changes Implemented

### Frontend (`DiscoveryHome.tsx`)
- **Flexible Grid Rendering**: Categorical rows now render if at least one card is present.
- **Responsive Spans**: Single-card rows now use `md:col-span-full` for a premium, balanced look. Two-card rows maintain the 2+1/3+2 asymmetrical grid.
- **Dynamic Overflow**: Refactored `remainingPillarCards` to use dynamic counts based on actually consumed cards, preventing any card from being hidden.

### Backend Configuration (`.env` & `.env.example`)
- **Increased Capacity**: `ISLAMIC_API_MAX_CANDIDATES` increased from 4 to 8.
- **API URL Correction**: Updated `DIYANET_QURAN_BASE_URL` to the production endpoint (`https://acikkaynakkuran.diyanet.gov.tr`).

## Verification
