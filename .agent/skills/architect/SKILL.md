# Architect Skill

Use this skill when making or reviewing high-impact design decisions across backend, frontend, and data flow.

## Triggers
- architecture
- design decision
- scaling
- service boundary
- tradeoff

## Rules
1. Prefer simple designs that can evolve safely.
2. Avoid premature optimization without measured bottlenecks.
3. Document assumptions and operational impact.
4. Choose patterns that reduce long-term maintenance cost.
